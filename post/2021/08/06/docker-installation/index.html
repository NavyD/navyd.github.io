Manage Docker as a non-root user

在安装完成后不能使用非root用户运行，可能遇到下面的问题：

1
2

$ docker stats
Got permission denied while trying to connect to the Docker daemon socket at unix:///var/run/docker.sock: Get "http://%2Fvar%2Frun%2Fdocker.sock/v1.24/version": dial unix /var/run/docker.sock: connect: permission denied

可以为当前用户添加到docker组中解决

1
2
3
4

sudo groupadd docker
sudo usermod -aG docker "$USER"
#  Change user's primary group membership or logout and login again or reboot
newgrp docker

Rootless mode

在 Run the Docker daemon as a non-root user (Rootless mode)中可以不作为root用户运行，在Docker Engine v20.10发布

在ubuntu上可以直接运行dockerd-rootless-setuptool.sh install或下载运行curl -fsSL https://get.docker.com/rootless | sh

配置wsl2 docker自启动

wsl2没有systemd，不能使用官方教程 Configure Docker to start on boot，只能使用service启动。

配置sudo在运行service命令时不会请求密码

1

echo "$USER ALL=(ALL:ALL) NOPASSWD: /usr/sbin/service" | sudo tee -a "/etc/sudoers.d/$USER"

然后在~/.zshrc中启动docker

1

service docker status > /dev/null || sudo service docker start

  • sudo visudo 退出保存

docker更换国内镜像

配置文件启动Docker,修改/etc/docker/daemon.json

1
2
3
4
5
6

{
    "registry-mirrors": [
        "http://hub-mirror.c.163.com/",
        "https://docker.mirrors.ustc.edu.cn/"
    ]
}

修改保存后，重启 Docker 以使配置生效。docker info查看

1
2
3
4
5
6

$ sudo service docker restart

$ docker info | grep -i mirror
 Registry Mirrors:
  http://hub-mirror.c.163.com/
  https://docker.mirrors.ustc.edu.cn/
